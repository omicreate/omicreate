# 公開・更新チェックリスト

omicreateで新しいリポジトリやアプリを公開するとき、または公開済みリポジトリのREADMEや公開ページを更新するときに確認する項目です。

## README

- [ ] `docs/README_TEMPLATE.md` の構成に沿っている
- [ ] 概要、公開URL、主な機能、使い方が最初に読める
- [ ] `データ/プライバシー` を書いている
- [ ] `権利/ライセンス` を書いている
- [ ] 公開URLとリポジトリURLが正しい
- [ ] 相対リンクが壊れていない

## ローカル情報・秘密情報

README、公開ドキュメント、ソースコードに以下が含まれていないことを確認します。

- [ ] 個人端末のホームディレクトリを示す絶対パス
- [ ] `C:\`
- [ ] `.env` の中身
- [ ] 開発用のループバックホスト名
- [ ] 開発用のループバックIPアドレス
- [ ] `API_KEY`
- [ ] `SECRET`
- [ ] `TOKEN`
- [ ] 端末名、OSユーザー名、作業ディレクトリ
- [ ] privateリポジトリ名、未公開URL、内部運用メモ
- [ ] 公開許可を取っていない個人情報、実データ、スクリーンショット

## ライセンス

- [ ] コードのライセンスを決めてREADMEに明記している
- [ ] MIT Licenseの場合はルートに `LICENSE` を置いている
- [ ] All rights reservedの場合は `LICENSE.md` または `TERMS.md` で再利用条件を明記している
- [ ] コードと素材の扱いが違う場合、READMEで分けて説明している
- [ ] 他者素材、外部ライブラリ、フォント、画像を使う場合、利用条件に反していない

## データ/プライバシー

- [ ] アカウント登録の有無を説明している
- [ ] 入力データの保存場所を説明している
- [ ] 外部送信、外部API、アクセス解析の有無を説明している
- [ ] 個人名、学校名、チーム名などを入力し得る場合、利用者側の管理責任や注意点を書いている
- [ ] `PRIVACY.md` が必要なアプリでは作成している

## 公開・更新時

- [ ] READMEをGitHub表示で確認した
- [ ] 公開URLを実際に開いて確認した
- [ ] 主要操作を一通り試した
- [ ] OGP画像やスクリーンショットに非公開情報が写っていない
- [ ] 公開範囲が意図どおり public / private になっている
