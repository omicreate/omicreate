# READMEテンプレート

このテンプレートは、omicreateで今後公開するプロジェクトのREADMEをそろえるための基準です。
公開時や更新時に `docs/PUBLIC_RELEASE_CHECKLIST.md` も確認してください。

## 使い方

- 新規公開リポジトリの `README.md` は、この構成を元に作成します。
- アプリの性質に合わせて見出しを増減して構いませんが、`データ/プライバシー` と `権利/ライセンス` は必ず残します。
- READMEには公開してよい情報だけを書き、ローカル環境や秘密情報は載せません。

## テンプレート

```md
# プロジェクト名

1〜3文で、何をするものか、誰に向けたものか、ブラウザ/スマホ/PCなどの利用環境を説明します。

## 公開URL

- 使う: https://omicreate.github.io/REPOSITORY_NAME/
- リポジトリ: https://github.com/omicreate/REPOSITORY_NAME

## 主な機能

- 機能1
- 機能2
- 機能3

## 使い方

1. 公開URLを開きます。
2. 最初に行う操作を書きます。
3. 結果の確認方法や共有方法を書きます。

## データ/プライバシー

- アカウント登録の有無を書きます。
- 入力データの保存場所を書きます。
- 外部送信、外部API、アクセス解析の有無を書きます。
- 利用者が個人情報を入力する可能性がある場合は、扱い方の注意を書きます。

## 権利/ライセンス

- コード: MIT License / All rights reserved / 未設定 のいずれかを明記します。
- 画像・ロゴ・文章・キャラクター・音声など、コード以外の素材の扱いを明記します。
- 詳細ファイルがある場合は `LICENSE`、`LICENSE.md`、`TERMS.md`、`PRIVACY.md`、`SECURITY.md` にリンクします。

## 技術構成

- 実行方式: 例）静的HTML / Vite / Next.js など
- 主なファイル: 例）`index.html`, `app/script.js`
- 外部ライブラリや外部APIを使う場合は、公開してよい範囲で書きます。

## 確認方法

```sh
npm test
```

ビルド不要の場合は、ブラウザで開いて確認する項目を書きます。
```

## READMEに載せない情報

- ローカル絶対パス、端末名、OSユーザー名、作業ディレクトリ
- `.env` の内容、APIキー、トークン、シークレット、認証情報
- 未公開URL、privateリポジトリ名、内部運用メモ
- 個人情報、実在の利用者データ、公開許可を取っていないスクリーンショット

## ライセンスの基本方針

- 実用アプリ、独自性の高い分析ロジック、画面設計、文章、画像は原則として権利保護寄りにします。
- 再利用されてもよい小さなコード、学習用サンプル、タイピングゲームのコードは原則としてMIT Licenseにします。
- コードと素材の扱いが違う場合は、READMEで必ず分けて書きます。
